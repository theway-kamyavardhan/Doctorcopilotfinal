# DoctorCopilot: Full Stack Project Architecture & Overview

DoctorCopilot is a modern, AI-driven medical intelligence platform. It is split into a decoupled Frontend (Client) and Backend (API) architecture. 

---

## 🏗️ 1. Full Stack Framework Overview

### 🎨 Frontend (The User Interface)
- **Framework:** React 18 + Vite (for ultra-fast bundling and HMR).
- **Styling:** TailwindCSS (for utility-first styling) + Framer Motion (for smooth micro-animations).
- **Design System:** Apple-inspired minimalist Glassmorphism (using `backdrop-blur` and dynamic `ThemeContext` resolution).
- **Routing:** React Router DOM.
- **Charts/Data Vis:** Recharts (for medical timelines and tracking hemoglobin/platelet trends).
- **State Management:** React Context API + LocalSession Cache (e.g. `PatientDashboard` 5-minute cache) + Local React State.
- **Authentication:** JWT (JSON Web Tokens) stored in localStorage, intercepted via Axios.

### 👥 Role-Based Portals
1. **Patient Portal**: The default dashboard for uploaded and extracted medical history, timelines, and anomaly streams. 
2. **Admin Control Surface**: An ultra-minimalistic, glassmorphic realtime matrix for system monitoring, user administration, and OCR pipeline checks.
3. **Doctor Workspace**: A futuristic, 3-panel triage layout featuring chronological patient history, an AI Clinical Engine for automated case interpretation, and interactive chat routing.

### ⚙️ Backend (The Server & AI Engine)
- **Framework:** FastAPI (Python 3.11+). Chosen for its incredibly fast performance, async support, and native OpenAPI/Swagger documentation.
- **Database ORM:** SQLAlchemy (sync/async database abstraction layer).
- **Data Validation:** Pydantic (ensures data coming in/out of the API matches strict medical schemas).
- **Database:** SQLite (Default for quick local development) / PostgreSQL (Supabase/Neon for production).
- **AI Integration:** Google Gemini (used for clinical normalization, extracting structured data from unstructured OCR text, and AI medical summaries).

## 👁️ 2. The OCR Engine: Tesseract & Requirements
To read scanned medical PDFs, DoctorCopilot relies on a powerful Optical Character Recognition (OCR) pipeline before AI analysis.

**Core Dependencies for Document Processing:**
1. **Tesseract OCR (`tesseract-ocr`)**: 
   - This is NOT a Python library. It is a system-level C++ software developed by Google. It actually "looks" at images and extracts the English text.
   - **Requirement:** Must be installed on the host operating system (e.g., via `apt-get install tesseract-ocr` or a Windows installer) for the backend to work.
2. **`pytesseract`**: The Python wrapper that allows our FastAPI backend to talk to the Tesseract software installed on the computer.
3. **`pdf2image`**: Converts uploaded multipage PDF medical reports into images that Tesseract can read.
4. **`poppler-utils`**: A system dependency required by `pdf2image` to handle the actual PDF translation.

*Flow*: User uploads PDF -> Backend uses `poppler` to render images -> `Tesseract` reads text -> `Gemini` structures the text into JSON.

## 🚀 4. How to Run It Locally (Quick Reference)

**Start the Backend:**
1. Open a terminal.
2. Activate your virtual environment: `.\venv\Scripts\activate` (Windows)
3. Start FastAPI server: `uvicorn app.main:app --reload`
4. The API (and Swagger Docs) are now live at `http://127.0.0.1:8000/docs`

**Start the Frontend:**
1. Open a second terminal.
2. Run Vite: `npm run dev`
3. The UI is live at `http://localhost:5173`
