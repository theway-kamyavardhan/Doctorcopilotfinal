DOCTORCOPILOT GUIDE PACK - START HERE

Purpose
- This file tells you which guide to read first.
- It turns the scattered text files in the guide folder into a practical step-by-step reading order.

Best Reading Order

1. If you are completely new
- Read `vocabulary.txt`
- Then read `introduction.txt`
- Then read `STEP_BY_STEP_SYSTEM_WALKTHROUGH.txt`

2. If you want to run the project locally
- Read `STEP_BY_STEP_LOCAL_RUN.txt`

3. If you want to deploy the project online
- Read `STEP_BY_STEP_DEPLOYMENT.txt`

4. If you want to understand doctor-side architecture in more detail
- Read `doctor.txt`

5. If you want the architecture summary
- Read `DoctorCopilot_Architecture.txt`
- Then read `moon.txt`

6. If you want migration help
- Read `supabase_migration.txt`
