DoctorCopilot Local -> Supabase One-Time Migration

Purpose
- Copy the existing local SQLite data into Supabase Postgres.
- Copy the existing uploaded report files into the Supabase Storage bucket.
- Preserve extracted report data and AI results as-is.
- Avoid re-running OCR or AI processing.

Script
- app/scripts/migrate_local_to_supabase.py

What it migrates
- users
- patients
- doctors
- cases
- messages
- appointments
- clinical_notes
- reports
- extracted_data
- report_insights
- processing_logs

What it does with files
- Reads the original file referenced by each local report row.
- Uploads that original file to the configured Supabase bucket.
- Stores the bucket object path in Postgres.
- Does not reprocess the report.

Recommended run
- Use --reset-target when you want Supabase to become an exact copy of the local app state.

Example PowerShell command
- Set ENVIRONMENT=production
- Set DATABASE_URL to the Supabase session pooler URL
- Set SUPABASE_URL
- Set SUPABASE_SERVICE_ROLE_KEY
- Set SUPABASE_STORAGE_BUCKET=reports
- Run:
  python -m app.scripts.migrate_local_to_supabase --reset-target

Notes
- Rotate any leaked service-role keys before deployment.
- Keep the local SQLite database and local uploads as a backup until production is confirmed.
